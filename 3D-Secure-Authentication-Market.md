<h2 data-start=""1314"" data-end=""1356""><strong data-start=""1317"" data-end=""1354"">What is 3D Secure Authentication?</strong></h2>
<p data-start=""1358"" data-end=""1630""><strong data-start=""1358"" data-end=""1377"">3D Secure (3DS)</strong> is a security protocol that adds an extra layer of authentication to online transactions made with <strong data-start=""1477"" data-end=""1503"">credit and debit cards</strong>. It is designed to <strong data-start=""1523"" data-end=""1608"">reduce fraud, prevent unauthorized transactions, and increase consumer confidence</strong> in online payments.</p>
<h3 data-start=""1632"" data-end=""1666""><strong data-start=""1636"" data-end=""1664"">How Does 3D Secure Work?</strong></h3>
<ol data-start=""1667"" data-end=""2144"">
<li data-start=""1667"" data-end=""1762""><strong data-start=""1670"" data-end=""1701"">Customer Initiates Payment:</strong> The user enters card details during an online transaction.</li>
<li data-start=""1763"" data-end=""1867""><strong data-start=""1766"" data-end=""1793"">Authentication Request:</strong> The transaction is redirected to the card issuer&rsquo;s authentication page.</li>
<li data-start=""1868"" data-end=""2034""><strong data-start=""1871"" data-end=""1893"">Verification Step:</strong> The customer verifies their identity using a <strong data-start=""1939"" data-end=""2031"">password, one-time password (OTP), biometric authentication, or app-based authentication</strong>.</li>
<li data-start=""2035"" data-end=""2144""><strong data-start=""2038"" data-end=""2076"">Transaction Approval or Rejection:</strong> If verified, the transaction is approved; if not, it is declined.</li>
</ol>
<p data-start=""2146"" data-end=""2236"">The name <strong data-start=""2155"" data-end=""2170"">""3D Secure""</strong> refers to three domains involved in the authentication process:</p>
<ol data-start=""2237"" data-end=""2480"">
<li data-start=""2237"" data-end=""2291""><strong data-start=""2240"" data-end=""2258"">Issuer Domain:</strong> The bank that issued the card.</li>
<li data-start=""2292"" data-end=""2369""><strong data-start=""2295"" data-end=""2315"">Acquirer Domain:</strong> The merchant's bank that processes the transaction.</li>
<li data-start=""2370"" data-end=""2480""><strong data-start=""2373"" data-end=""2401"">Interoperability Domain:</strong> The infrastructure that facilitates authentication (Visa, Mastercard, etc.).</li>
</ol>
<h2 data-start=""2487"" data-end=""2535""><strong data-start=""2490"" data-end=""2533"">Evolution of 3D Secure: From 1.0 to 2.0</strong></h2>
<h3 data-start=""2537"" data-end=""2567""><strong data-start=""2541"" data-end=""2565"">3D Secure 1.0 (3DS1)</strong></h3>
<ul data-start=""2568"" data-end=""2846"">
<li data-start=""2568"" data-end=""2672"">Introduced in the early 2000s by <strong data-start=""2603"" data-end=""2669"">Visa (Verified by Visa) and Mastercard (Mastercard SecureCode)</strong>.</li>
<li data-start=""2673"" data-end=""2742"">Required customers to enter a static password for authentication.</li>
<li data-start=""2743"" data-end=""2846""><strong data-start=""2745"" data-end=""2759"">Drawbacks:</strong> High cart abandonment rates, poor user experience, and limited mobile compatibility.</li>
</ul>
<h3 data-start=""2848"" data-end=""2878""><strong data-start=""2852"" data-end=""2876"">3D Secure 2.0 (3DS2)</strong></h3>
<ul data-start=""2879"" data-end=""3265"">
<li data-start=""2879"" data-end=""2940"">Launched in <strong data-start=""2893"" data-end=""2901"">2017</strong> to address the shortcomings of 3DS1.</li>
<li data-start=""2941"" data-end=""3054"">Introduced <strong data-start=""2954"" data-end=""2983"">risk-based authentication</strong>, which only prompts authentication if a transaction is deemed risky.</li>
<li data-start=""3055"" data-end=""3148"">Supports <strong data-start=""3066"" data-end=""3118"">biometrics, tokenization, and device recognition</strong>, enhancing user experience.</li>
<li data-start=""3149"" data-end=""3265"">Fully compliant with <strong data-start=""3172"" data-end=""3225"">Strong Customer Authentication (SCA) requirements</strong> under <strong data-start=""3232"" data-end=""3262"">PSD2 regulations in Europe</strong>.</li>
</ul>
